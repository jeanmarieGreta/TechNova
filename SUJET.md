# Sujet — Épreuve de synthèse TSSR

## Mise en situation professionnelle

Vous intervenez en tant qu’**équipe technique** au sein d’une entreprise qui souhaite mettre en place une **suite collaborative interne**.

La direction souhaite disposer d’une solution permettant :

- la **gestion centralisée des utilisateurs**
- le **partage de documents**
- la **communication interne**
- la **visioconférence**
- une **administration structurée**
- une **première sécurisation des accès**

La solution technique étudiée repose sur les briques suivantes :

- **Active Directory / DNS / DHCP**
- **Nextcloud**
- **Rocket.Chat**
- **Jitsi Meet**

Vous êtes chargés de réaliser un **POC** afin de valider la faisabilité technique, puis de préparer les éléments nécessaires à une **intégration finale**.

---

## Composition de l’équipe

Le travail est réalisé par **3 stagiaires**.

Vous devez vous organiser comme une **petite équipe d’intégration**.

Une répartition des rôles est attendue, mais chaque membre de l’équipe doit être capable de comprendre et d’expliquer l’architecture globale.

---

## Mission

Votre mission comporte **deux volets**.

## 1. Réalisation du POC

Vous devez mettre en place un environnement technique de démonstration permettant de valider les points essentiels de la solution.

Le POC doit permettre de montrer que :

- l’infrastructure de base est opérationnelle ;
- les services retenus peuvent être déployés ;
- les composants communiquent correctement ;
- la solution est cohérente d’un point de vue technique.

## 2. Préparation de l’intégration finale

À partir du POC réalisé, vous devez identifier les actions à mener pour passer à une intégration plus propre, plus fiable et mieux sécurisée.

Vous devez donc distinguer :

- ce qui relève du **prototype fonctionnel**
- ce qui relève de la **solution prête à être intégrée**

---

## Travail demandé

## Partie A — Analyse rapide

Avant toute mise en œuvre, vous devez :

1. prendre connaissance de l’architecture cible ;
2. identifier les dépendances entre les services ;
3. répartir les rôles au sein de l’équipe ;
4. définir un ordre logique de travail.

## Partie B — Mise en œuvre du POC

Vous devez :

1. déployer l’infrastructure nécessaire ;
2. configurer les services principaux ;
3. vérifier leur bon fonctionnement ;
4. réaliser une démonstration technique minimale ;
5. relever les difficultés rencontrées.

## Partie C — Préparation de l’intégration finale

Vous devez ensuite produire une analyse simple indiquant :

1. ce qui fonctionne dans le POC ;
2. ce qui reste fragile ou incomplet ;
3. ce qu’il faudrait modifier avant intégration finale ;
4. les mesures minimales de sécurité à prévoir ;
5. les tests complémentaires à réaliser.

---

## Livrables attendus

À la fin du travail, vous devez fournir :

## Livrables techniques

- un **POC fonctionnel**
- une **démonstration** des services retenus
- une **validation minimale** du fonctionnement

## Livrables d’analyse

- une **liste des écarts** entre le POC et l’intégration finale
- une **liste des points de vigilance**
- une **proposition de suite logique pour l’intégration**

## Livrables oraux

- une **présentation collective courte**
- une **explication claire des choix techniques**
- une **justification des limites du POC**

---

## Contraintes

- travail en **équipe de 3**
- restitution **collective**
- chaque stagiaire doit connaître **son rôle** et **le projet global**
- tout élément annoncé comme fonctionnel doit pouvoir être **montré**
- tout choix technique doit pouvoir être **justifié**
